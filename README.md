# SDD Learning Bundle

**Spec-Driven Development** — An AI-native methodology that makes delivery metrics move.

## What's Inside

### Methodology
- `CLAUDE.md` — Complete SDD methodology reference, project conventions, and development workflow

### Pipeline Agents
- `.claude/agents/sdd/pm-spec.md` — Spec authoring and structuring
- `.claude/agents/sdd/architect-review.md` — Architecture validation
- `.claude/agents/sdd/implementer-tester.md` — Implementation coordination and QA
- `.claude/agents/sdd/deployment.md` — Deployment preparation
- `.claude/agents/sdd/learning-engine.md` — Learning event analysis

### Process & Quality
- `.claude/qa-checklist.md` — QA checklist template (run before every commit)
- `.claude/development-workflow.md` — Complete development workflow

### Governance
- `governance/solo-operator-model.md` — Gate ownership for solo operators
- `governance/tier-selection-guidelines.md` — Complexity tier selection guide
- `governance/escalation-protocols.md` — When and how to escalate tier
- `governance/stack-quirks.md` — Platform and tool gotchas

### Spec Templates (SDD Experimental)
- `specs/templates/gate-annotation-guide.md` — How to annotate gate decisions
- `specs/templates/post-completion-retro.md` — Post-deployment retrospective template

### Learning Library
Real learning events from production use of SDD:
- `learning/process-learnings-2026-02.md` — Monthly process observations
- `learning/catches/` — Pre-production catches (4 events)
- `learning/escapes/` — Production escapes with root cause analysis (5 events)
- `learning/process-gaps/` — Process gap identification (3 events)

## Getting Started

1. Copy `CLAUDE.md` to your project root
2. Copy `.claude/` directory to your project
3. Adapt `governance/` documents to your team structure
4. Start your first spec with the PM-Spec agent
